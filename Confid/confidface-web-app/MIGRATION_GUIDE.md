# Migration from Akool to D-ID Knowledge Base

## Overview
This project has been successfully migrated from Akool's streaming avatar SDK to D-ID's /talks streaming API with the following architecture:

✅ **Knowledge Management**: Convex + your backend  
✅ **Text Generation**: Google Gemini AI  
✅ **Avatar Streaming**: D-ID /talks API  

## What Changed

### 1. **API Routes**
- `app/api/akool-knowledge-base/` → `app/api/d-id-knowledge-base/`
- `app/api/akool-session/` → `app/api/d-id-session/`

### 2. **Knowledge Base Management**
**Before (Akool):**
- Used Akool's cloud knowledge base API
- Questions stored on Akool servers

**After (D-ID + Gemini):**
- Knowledge managed through Convex + your backend
- Gemini generates conversational responses based on interview questions
- D-ID handles avatar streaming only

### 3. **Avatar Streaming**
**Before (Akool):**
- Used `akool-streaming-avatar-sdk` with Agora RTC
- Required Akool session with knowledge base ID

**After (D-ID):**
- Uses D-ID's /talks streams API with WebRTC
- Direct peer-to-peer connection
- More control over conversation flow

### 4. **Dependencies**
**Removed:**
- `akool-streaming-avatar-sdk`

**Added:**
- `@google/generative-ai` - For Gemini text generation

## Setup Instructions

### 1. Install Dependencies
```bash
npm install
```

### 2. Configure Environment Variables
Create a `.env.local` file with the following:

```env
# D-ID Configuration
D_ID_API_KEY=your_d_id_api_key
NEXT_PUBLIC_D_ID_API_KEY=your_d_id_api_key

# Gemini Configuration
GEMINI_API_KEY=your_gemini_api_key
NEXT_PUBLIC_GEMINI_API_KEY=your_gemini_api_key

# Keep your existing Convex and other configs
```

### 3. Get API Keys

**D-ID API Key:**
1. Sign up at [D-ID Studio](https://studio.d-id.com/)
2. Go to Account Settings
3. Create an API key
4. Copy and paste into `.env.local`

**Gemini API Key:**
1. Visit [Google AI Studio](https://makersuite.google.com/app/apikey)
2. Create a new API key
3. Copy and paste into `.env.local`

### 4. Choose D-ID Presenter
The default presenter is `amy-jcwCkr1grs`. To use a different avatar:
1. Browse [D-ID Presenters](https://docs.d-id.com/reference/presenters)
2. Update `PRESENTER_ID` in `app/(routes)/interview/[interviewId]/start/page.tsx`

## Architecture Flow

```
1. User starts interview
   ↓
2. Convex fetches interview questions from your backend
   ↓
3. Questions sent to d-id-knowledge-base API
   ↓
4. Gemini generates initial greeting & first question
   ↓
5. D-ID session created with generated text
   ↓
6. WebRTC connection established
   ↓
7. Avatar streams video with speech
   ↓
8. User responds via microphone
   ↓
9. Response processed → Gemini generates next question
   ↓
10. Cycle continues until interview ends
```

## Troubleshooting

### Video not loading?
- Check `D_ID_API_KEY` is set correctly
- Verify presenter ID exists
- Check browser console for WebRTC errors

### No audio from avatar?
- Ensure browser allows audio playback
- Check D-ID API response for errors
- Verify script text is being generated by Gemini

### Gemini not generating responses?
- Check `GEMINI_API_KEY` is valid
- Verify API quota not exceeded
- Check console for API errors
